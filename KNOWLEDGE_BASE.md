# База знаний — MetalCalc (Металлоёмкость зданий)

> Версия: 1.0 · Дата: 2026-03-14

---

## 1. Назначение

Программа рассчитывает ориентировочную металлоёмкость стального каркаса производственных зданий по двум методам (М1 и М2) согласно методике «Определение расхода стали на несущие конструкции каркаса», ЦНИИ Строительных конструкций.

## 3. Конструктивные элементы и методика расчёта

### 3.1 Прогоны

**Метод:** подбор по таблице 3 методики.

```
qp = (Q_roof + Q_purlin + Q_snow + Q_dust + Q_tech) · a_пр · γn / 9.81  [т/м]
```
где `a_пр` — шаг прогонов (принимается = шагу ферм B).

**Таблица прогонов (PURLIN_TABLE):**
| qp_max (т/м) | Профиль B=6м | Масса 6м (кг) | Профиль B=12м | Масса 12м (кг) |
|---|---|---|---|---|
| 0.45 | Швеллер 20 | 110.4 | Двутавр 20 | 345.0 |
| 0.67 | Швеллер 22 | 122.4 | Двутавр 24 | 420.0 |
| 0.87 | Швеллер 24 | 134.4 | Двутавр 27 | 492.0 |
| 1.10 | Швеллер 27 | 151.2 | Двутавр 30 | 558.0 |
| 1.30 | Швеллер 30 | 168.0 | Двутавр 33 | 624.0 |
| 1.50 | Швеллер 33 | 184.8 | Двутавр 36 | 697.2 |

Расход: `g_пр = M_прогона · n_прогонов / (L · B)` [кг/м²]

---

### 3.2 Фермы стропильные

**Метод 1 (Уголки):** `g_ф = tabulated_kg/m² · S_пролёта`

**Метод 2 (Двутавры, Молодечно):** таблицы заводов по нагрузке `Q_load_total` и пролёту L.

Таблица `TRUSS_MASSES[тип][пролёт][индекс_нагрузки]` — кг/м²
Интерполяция: ceiling по ближайшей нагрузке из `TRUSS_LOADS`.

Пролёты: 18, 24, 30, 36 м. Нагрузки: от 2.0 до 12.5 кН/м² с шагом 0.5.

---

### 3.3 Фермы подстропильные

Применяются при шаге колонн `col_step = 12 м`.

Реакция на подстропильную ферму:
```
R = Q_total · L_span / 2 · (col_step / B_step)  [кН]
```
Масса подбирается по таблице `SUBTRUSS_LOADS / SUBTRUSS_MASSES` (кг/м²).

---

### 3.4 Связи по покрытию

```
g_св = BRACING_TABLE[q_crane_t][B_step]  [кг/м²]
```

| Кран | B=6м | B=12м |
|---|---|---|
| ≤ 120 т | 15 кг/м² | 35 кг/м² |
| > 120 т | 40 кг/м² | 55 кг/м² |

### 3.5 Подкрановые балки (М1 и М2)

#### М1 — аналитическая формула

```
Gпб,n = (αпб × Lпб + qр) × Lпб × kпб   [кН]
Gпб [т] = Gпб,n / 9.81 × mf_режим
g_пб = Gпб · 1000 / S_площадь  →  расход кг/м²
```

где `kпб = 1.2` — конструктивный коэффициент (тормозная конструкция + связи + крепление рельса).

> Источник: методика ЦНИИ, разд. «Подкрановые балки».
> Контрольный пример: Q=100/20т, L=6м → Gпб = (0.39×6+1.13)×6×1.2 = 24.98 кН = 2498 кг (g=10).

Коэффициент `αпб` (`CRANE_BEAM_ALPHA`) — линейная интерполяция по диапазонам методики:
- Q = 20–50 т: αпб = 0.24 – 0.35
- Q = 80–200 т: αпб = 0.37 – 0.47
- За пределами диапазона: линейная экстраполяция

| Q, т | 5 | 10 | 20 | 32 | 50 | 80 | 100 | 125 | 200 | 320 | 400 |
|---|---|---|---|---|---|---|---|---|---|---|---|
| αпб | 0.19 | 0.20 | 0.24 | 0.28 | 0.35 | 0.37 | 0.39 | 0.41 | 0.47 | 0.57 | 0.64 |

`CRANE_Q_EQUIV` в расчёте подкрановых балок **не используется** — только в расчёте колонн для `Dmax`.

Эквивалентная нагрузка `qэкв` (`CRANE_Q_EQUIV`, кН/м) — только для колонн:
| Q, т | 5 | 10 | 20 | 32 | 50 | 80 | 100 | 125 | 200 | 320 | 400 |
|---|---|---|---|---|---|---|---|---|---|---|---|
| qэкв | 20 | 30 | 50 | 70 | 95 | 138 | 170 | 200 | 262 | 362 | 437 |

Масса рельса `RAIL_WEIGHT_KN` (кН/м):
| Q, т | ≤20 | ≤50 | ≤80 | ≤125 | ≤200 | ≤400 |
|---|---|---|---|---|---|---|
| кН/м | 0.461 | 0.598 | 0.831 | 1.135 | 1.135 | 1.417 |

#### М2 — табличный метод (CRANE_BEAM_T1 / T2)

Таблицы `CRANE_BEAM_T1` (Q ≤ 50 т) и `CRANE_BEAM_T2` (Q ≥ 80 т) — масса балки в кг/м.

#### Коэффициент режима работы крана

| Режим | М1 | М2 |
|---|---|---|
| 1-6К (лёгкий/средний) | 1.00 | 0.65 |
| 7-8К (тяжёлый) | **1.80** | **1.80** |

> **Важно:** коэффициент 1.80 для 7-8К — скорректированное значение по результатам калибровки (было 1.15). Разница обоснована тем, что базовая формула М1 откалибрована на режим 1-6К.

### 3.7 Колонны

#### Расчётная схема (аналитическая)

Нижняя часть колонны воспринимает:
- вертикальные нагрузки от покрытия, снега, кровли
- вес подкрановой балки + рельс
- нагрузку от мостового крана Dmax
- собственный вес верхней части колонны

Верхняя часть колонны воспринимает только нагрузки от покрытия.

#### Формулы

```
# Верхняя часть (надкрановая)
SFv = Q_load · cs · L / 2 + Gwu
Gcu = SFv · ρ · ψу · Hн / (kMu · Ry)

# Нижняя часть (подкрановая)
Dmax = qэкв · Lпб + Gрельс + Gпб·αпб·Lпб
SFn = SFv + D + Gpb + Gwl + Gcu
Gcl = SFn · ρ · ψн · Hк / (kMl · Ry)
```

Параметры:

| Параметр | Значение | Пояснение |
|---|---|---|
| ρ (rho) | 78.5 кН/м³ | Удельный вес стали |
| ψу (pu) | 1.4 | Коэффициент заполнения верхней части |
| ψн (pl) | 2.1 | Коэффициент заполнения нижней части |
| kMu | 0.275 | Коэффициент использования момента (верх) |
| kMl | 0.45 | Коэффициент использования момента (низ) |
| Ry | **240 000 кН/м²** | Расчётное сопротивление С245: 24 кН/см² = 240 000 кН/м² |

> **Критическая ошибка, исправлена:** ранее Ry = 24 000 кН/м² (в 10 раз меньше) → масса колонн завышалась в ~10 раз. Исправлено на 240 000 кН/м².

#### Высота колонны

```
H_кол = h_rail + 4.5 м  (при H_col_ov = 0)
```
Разделение на верхнюю (надкрановую) и нижнюю части — по таблице `H_R_TABLES` в зависимости от грузоподъёмности крана.

Таблица `FAKHVERK_DATA` — расход кг/м² по ключу `(тип, индекс_высоты, индекс_нагрузки)`.

Тип фахверка:
- I: B_step = 6 м, без промежуточной стойки
- II: B_step = 12 м, без промежуточной стойки

## 6. Известные ограничения

1. **Однотипные краны**: программа предполагает одинаковые краны во всех пролётах (одна г/п). Смешанные краны не поддерживаются по-пролётно.
2. **Кратность пролётов**: поддерживаются только L = 18, 24, 30, 36 м.
3. **Подстропильные фермы**: только при col_step = 12 м.
4. **Метод М2**: доступен только при truss_type ∈ {Двутавры, Молодечно}.
5. **Фахверк**: рассчитывается только для торцевых стен.

## 7. Платформы и сборка

| Платформа | Файл | Инструмент |
|---|---|---|
| Windows | main_desktop.py | PyInstaller + CustomTkinter |
| macOS | main_desktop.py | PyInstaller + CustomTkinter |
| iOS | main.py | kivy-ios toolchain + Xcode |
| Android | main.py | Buildozer (не используется) |

### iOS: зависимости для сборки
- Python 3.11 (conda-env `kivy-ios`)
- kivy-ios (toolchain)
- Xcode 15+, Metal Toolchain
- brew: autoconf, automake, libtool, pkg-config, cmake

### iOS: известные проблемы KV-парсера
- `CLASS_RANGE` Kivy включает только A–Z. Класс `_Toolbar` не распознаётся как виджет (начинается с `_`) → переименован в `Toolbar`.
- Non-ASCII строки в KV-значениях → SyntaxError. Кириллица задаётся из Python после `build()`.
