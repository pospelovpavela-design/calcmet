# Как участвовать в проекте

Спасибо, что хочешь помочь! Ниже — правила совместной работы.

## Ветки

| Ветка | Назначение |
|-------|-----------|
| `main` | Стабильная версия, защищена. Прямые пуши запрещены |
| `dev` | Основная ветка разработки — сюда делать PR |
| `feature/<название>` | Новая функция |
| `fix/<название>` | Исправление бага |

## Рабочий процесс

```bash
# 1. Форкнуть репо и клонировать
git clone https://github.com/<твой-логин>/calcmet.git
cd calcmet

# 2. Создать ветку от dev
git checkout dev
git checkout -b feature/моя-фича

# 3. Работать, коммитить
git add .
git commit -m "feat: описание изменения"

# 4. Пушить и открывать PR в dev
git push origin feature/моя-фича
```

## Формат коммитов

```
feat: добавить расчёт колонн
fix: исправить деление на ноль при нулевом пролёте
docs: обновить README
refactor: вынести таблицы в отдельный модуль
test: добавить тест для PURLIN_TABLE
```

## Стиль кода

- Python 3.10+
- Отступы: 4 пробела
- Длина строки: до 100 символов
- Имена переменных и функций: `snake_case`
- Константы-таблицы: `UPPER_CASE`
- Комментарии можно на русском

## Оформление PR

- PR открывать **в ветку `dev`**, не в `main`
- Заполнить шаблон PR (появится автоматически)
- Добавить скриншот если меняется UI
- Один PR = одна задача

## Баги и идеи

Используй [Issues](../../issues) с шаблонами:
- **Bug report** — нашёл ошибку
- **Feature request** — хочешь новую функцию
